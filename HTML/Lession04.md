## Semantic Structure of a Web Page

- We have a structure in a web page and it is important because it helps browsers, developers, and screen readers understand which part of the page does what.
- Instead of using only `div` tags everywhere, HTML provides semantic tags that describe the purpose of a section.

### Header

- `header`
- Contains introductory content.
- Usually includes logo, title, search bar, etc.

### Navigation Bar

- `nav`
- Contains navigation links used to move around the website.

### Main Content

- `main`
- Contains the primary content of the webpage.
- A page should generally have only one `main` tag.

#### Sub-sections inside Main

##### Article Tag

- `article`
- Represents a self-contained piece of content.
- Example: Blog post, news article, forum post.

##### Section Tag

- `section`
- Represents a thematic section of a page.
- Used to group related content together.

##### Div Tag

- `div`
- Generic container.
- Used when no semantic tag fits the purpose.

### Sidebar

- `aside`
- Contains content related to the main content but not part of it.
- Example: Advertisements, related posts, quick links, author info.

### Footer

- `footer`
- Usually contains copyright information, contact details, social links, etc.

---

### Why use Semantic Tags?

- Better code readability.
- Better accessibility (screen readers can understand the page structure).
- Better SEO (search engines can better understand the content).
- Easier maintenance and collaboration.
